# Protocol Architecture

AnchorVault coordinates three distinct entities trustlessly on-chain: **Liquidity Providers**, **Payment Anchors**, and the **Core Smart Contracts**.

## Smart Contract Topology

The protocol consists of **four on-chain contracts** that interact in a coordinated manner:

```mermaid
graph LR
    classDef core fill:#7b39fc,stroke:#fff,color:#fff
    classDef token fill:#00e5ff,stroke:#fff,color:#000

    USDC["USDC Stablecoin<br/>(SAC Token)"]:::token
    AVLT["$AVLT Share Token<br/>(Vault Token)"]:::token
    Vault["Corridor Pool<br/>Core Vault"]:::core
    Registry["Anchor<br/>Registry"]:::core

    USDC -->|"deposit / withdraw"| Vault
    Vault -->|"mint / burn shares"| AVLT
    AVLT -->|"lock collateral"| Registry
    Registry -->|"whitelisting check"| Vault
    Vault -->|"draw / repay USDC"| USDC
    Vault -->|"reputation update"| Registry
```

| Contract | Responsibility |
|:---------|:---------------|
| **USDC Stablecoin** | Core asset used in all corridor pool operations. Casper Asset Contract (SAC). |
| **Vault Share Token ($AVLT)** | ERC20-like token representing LP ownership. Minted on deposit, burned on withdrawal. |
| **Corridor Pool Vault** | Central vault managing deposits, withdrawals, draws, repayments, and yield distribution. |
| **Anchor Registry** | Manages anchor whitelisting, collateral lockups, reputation tracking, and credit limits. |

## Full Operational Lifecycle

```mermaid
sequenceDiagram
    autonumber
    actor LP as Liquidity Provider
    actor Anchor as Payment Anchor
    participant Registry as Anchor Registry
    participant Vault as Corridor Pool Vault
    participant AVLT as $AVLT Token
    participant USDC as USDC Contract

    Note over LP, USDC: Phase 1 — Liquidity Provisioning
    LP->>Vault: deposit(lp_address, amount)
    Vault->>USDC: transfer(lp → vault, amount)
    Vault->>Vault: Calculate LP shares (pro-rata)
    Vault->>AVLT: mint(lp_address, share_amount)
    AVLT-->>LP: $AVLT Shares Credited

    Note over Anchor, Registry: Phase 2 — Anchor Onboarding
    Anchor->>Registry: register_anchor(admin, anchor, credit_limit)
    Anchor->>Registry: lock_collateral(amount)
    Registry->>AVLT: transfer(anchor → registry, amount)
    Registry->>Registry: Verify collateral ratio ≥ 10%

    Note over Anchor, Vault: Phase 3 — Settlement Draw
    Anchor->>Vault: draw_liquidity(anchor, amount)
    Vault->>Registry: assert is_whitelisted(anchor)
    Vault->>Vault: assert active_draw + amount ≤ credit_limit
    Vault->>USDC: transfer(vault → anchor, amount)
    Note over Anchor: Fulfills instant off-ramp payout

    Note over Anchor, Vault: Phase 4 — Repayment & Yield
    Anchor->>Vault: repay_liquidity(anchor, principal)
    Vault->>Vault: calculate_utilization() & fee_rate()
    Vault->>USDC: transfer(anchor → vault, principal + fee)
    Vault->>Vault: 90% fee → acc_fees_per_share
    Vault->>Vault: 5% fee → Insurance Fund
    Vault->>Registry: Boost reputation score
```

## Storage Architecture

The protocol uses Casper WASM's **instance** and **persistent** storage:

```mermaid
graph TD
    classDef instance fill:#4338CA,stroke:#fff,color:#fff
    classDef persistent fill:#7B39FC,stroke:#fff,color:#fff

    subgraph "Instance Storage (Global)"
        Pool["PoolState<br/>total_deposits, active_draws,<br/>reserve_balance, fee curve params"]:::instance
        InsuranceFund["InsuranceFund<br/>Accumulated safety reserves"]:::instance
        GovToken["GovernanceToken<br/>$AVLT contract address"]:::instance
    end

    subgraph "Persistent Storage (Per-Entity)"
        AnchorState["AnchorState(Address)<br/>credit_limit, active_draw,<br/>reputation_score"]:::persistent
        LPState["LPState(Address)<br/>shares, fee_debt"]:::persistent
    end
```

| Storage Type | Key | Data |
|:------------|:----|:-----|
| Instance | `Pool` | Global pool state — deposits, draws, reserves, fee parameters |
| Instance | `GovernanceToken` | Address of the $AVLT token contract |
| Instance | `InsuranceFund` | Accumulated insurance reserves (i128) |
| Persistent | `Anchor(Address)` | Per-anchor state — credit limit, draws, reputation |
| Persistent | `LP(Address)` | Per-LP state — shares owned, fee debt tracking |
